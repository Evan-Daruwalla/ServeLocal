Always-current snapshot of where the build stands. No history — for the chronological trail see

## What it is
A volunteer-opportunity platform: students discover/track community service, log verified hours, and
earn awards; organizations post opportunities and verify attendance. Personal/portfolio project.

## Architecture (as built)
- **Runtime:** near-zero dependencies — pure Node.js `http` (ADR-0001) plus one runtime dependency,
  `better-sqlite3` (ADR-0013, a documented exception, now required only inside `lib/persist.js`).
- **Frontend:** single-page SPA. Shell HTML/CSS in `public/index.html`; all JS extracted to
  `public/app.js` (ADR-0014, `<script src="/app.js" defer>`). Hash routing (`routeFromHash()`).
  Icons are self-hosted Twemoji PNGs; brand assets derived from one master logo.
- **Backend:** `server.js` — API routes as `if (method && path)` blocks, no router lib. Owns routing,
  static serving, and domain logic. Request pipeline: rate limit → auth (`getUser`, `tokenVersion`) →
  idempotency → handler; security headers on every response.
- **Extracted modules (ADR-0015):** `lib/persist.js` (SQLite/WAL mechanics), `lib/auth.js` (scrypt +
  HMAC-JWT + token hashing), `lib/totp.js` (RFC 6238 MFA). Pure mechanism, config injected via
  `init()`. `getUser`/`safeUser` stay in server.js (they need `IDX()`).
- **Persistence (ADR-0013 + incremental writes):** `db.sqlite` — one SQLite table per collection
  (`id`/`seq` primary key + a JSON `data` column per row), loaded fully into memory at boot into the
  same `DB.<collection>` arrays the rest of the codebase always used. `saveDB()` → `persist.save()`
  flushes **incrementally** over a persistent WAL handle (per-row sha1 mirror → only changed rows
  touch disk); fallback for first-boot/corrupt-recovery is temp-file + atomic rename. Format/engine
  swap only — the in-memory model, `IDX()` layer, and every handler are unchanged.
- **Indexes:** in-memory `IDX()` layer (ADR-0011) — primary-key + foreign-key `Map`s, rebuilt lazily
  on a structural-signature check (ADR-0012), not on every write. Plus `checkinCodeByStr` (O(1)
  check-in redemption, hint re-validated against the source of truth).
- **Pagination (ADR-0013):** `GET /api/opportunities` takes `limit` (default 60, max 200) / `offset`;
  response stays a bare array, total count travels in `X-Total-Count`. Discover has a "Load More" UI
  + "Showing X of Y", and mirrors filters into the URL hash (shareable/refresh-safe).
- **Auth:** stateless HMAC-JWT with `tokenVersion` revocation (ADR-0003); scrypt password hashing
  with transparent legacy migration (ADR-0009); self-service password reset (hashed single-use token,
  1h TTL, anti-enumeration); **TOTP MFA** (RFC 6238, two-step login, backup codes).
- **Security/ops:** hash-chained tamper-evident audit log (ADR-0005); in-memory rate limiting +
  circuit breaker (ADR-0006); public-read caching + gzip (ADR-0012); GDPR export/delete + retention
  purge; full controls matrix in `docs/security.md`.

## Known limitations
- **Billing is DEMO mode** — no real payments until Stripe keys are configured (ADR-0004).
- **In-memory-RAM ceiling not addressed** — every collection still loads fully into memory at boot;
  ADR-0013 fixed the *serialization* ceiling, not a hypothetical RAM one. The next step, if ever hit,
  is pushing queries down to SQL — a bigger, separate rewrite.
- ~~**CSP still allows `script-src 'unsafe-inline'`**~~ — **resolved (2026-07-05, ADR-0014 step 2):**
  all ~271 inline handlers converted to a delegated dispatch table (`ACTIONS` registry + one
  `document` listener per event type, `data-action`/`data-args` in the markup), so `script-src` is
  now `'self'`. `style-src` still allows `'unsafe-inline'` for inline `style=` attributes (out of
  scope).
- **One security item deferred** (`docs/security.md`): the HttpOnly-cookie auth refactor. (TOTP MFA
  shipped 2026-07-05; CAPTCHA/bot-defense on signup still deferred.)
- **Recurring-date math** mixes UTC and local time; needs a single-timezone anchor fix.
- **Audit log / verified hours / messages are intentionally not truncated** — bounding them at scale
  means archival-with-chain-preservation, a future ADR.
