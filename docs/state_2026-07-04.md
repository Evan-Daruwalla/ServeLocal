## What it is
A volunteer-opportunity platform: students discover/track community service, log verified hours, and
earn awards; organizations post opportunities and verify attendance. Personal/portfolio project.

## Architecture (as built)
- **Runtime:** near-zero dependencies — pure Node.js `http` (ADR-0001) plus one runtime dependency,
  `better-sqlite3` (ADR-0013, a documented exception). `servelocal` v2.0.0.
- **Frontend:** single-file SPA, all HTML/CSS/JS in `public/index.html`. Hash routing
  (`routeFromHash()`). Icons are self-hosted Twemoji PNGs; brand assets derived from one master logo.
- **Backend:** `server.js` — API routes as `if (method && path)` blocks, no router lib. Request
  pipeline: rate limit → auth (`getUser`, `tokenVersion`) → idempotency → handler; security headers
  on every response.
- **Persistence (ADR-0013):** `db.sqlite` — one SQLite table per collection (`id`/`seq` primary key +
  a JSON `data` column per row), loaded fully into memory at boot into the same `DB.<collection>`
  arrays the rest of the codebase always used. `saveDB()` builds a fresh sqlite file at a temp path
  then atomically renames it over the primary (temp+rename, same crash-safety guarantee as before).
  This is a persistence-*format* swap only — the in-memory model, `IDX()` layer, and every request
  handler are unchanged from before the migration.
- **Indexes:** in-memory `IDX()` layer (ADR-0011) — primary-key + foreign-key `Map`s, rebuilt lazily
  on a structural-signature check (ADR-0012), not on every write.
- **Pagination (ADR-0013):** `GET /api/opportunities` takes `limit` (default 60, max 200) / `offset`;
  response stays a bare array (no shape change), total count travels in `X-Total-Count`.
- **Auth:** stateless HMAC-JWT with `tokenVersion` revocation (ADR-0003); scrypt password hashing
  with transparent legacy migration (ADR-0009).
- **Security/ops:** hash-chained tamper-evident audit log (ADR-0005); in-memory rate limiting +
  circuit breaker (ADR-0006); GDPR export/delete + retention purge; full controls matrix in
  `docs/security.md`.

## Known limitations
- **Billing is DEMO mode** — no real payments until Stripe keys are configured (ADR-0004).
- **In-memory-RAM ceiling not addressed** — every collection still loads fully into memory at boot;
  ADR-0013 fixed the confirmed *serialization* ceiling, not a hypothetical RAM ceiling. The next step,
  if that's ever actually hit, is pushing queries down to SQL instead of loading everything into JS —
  a bigger, separate rewrite, not done here.
- **Frontend doesn't page past the default `limit=60`** on Discover — no "load more" UI was built
  (pagination shipped as a backend fix; a DB with more than 60 active listings only shows page 1
  until frontend paging is added).
- **Two security items intentionally deferred** (documented in `docs/security.md`): HttpOnly-cookie
  auth refactor and TOTP MFA. Not done — don't claim otherwise.
- **Recurring-date math** mixes UTC and local time; evening events in negative-UTC-offset timezones
  can key to the next UTC day. Needs a single-timezone anchor fix (don't patch piecemeal).
- **Audit log / verified hours / messages are intentionally not truncated** (records / tamper-
  evidence) — bounding them at scale means archival-with-chain-preservation, a future ADR.
