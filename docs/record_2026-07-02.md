## 2026-07-02 — Project inception  [reconstructed]

**WHAT:** Initial ServeLocal volunteer-platform codebase committed (`Initial commit: ServeLocal
volunteer platform`), followed by `Add README`. A zero-dependency Node.js web app: students find and
track community-service opportunities; orgs post them and verify hours.

**WHY:** Personal/portfolio full-stack project — chosen to be buildable end-to-end by one person and
to demonstrate engineering breadth (auth, persistence, security, tests, ops) without leaning on a
framework.

**HOW:** Deliberate zero-dependency constraint (later formalized as ADR-0001): pure Node `http`
module, a single-file SPA in `public/index.html`, and a JSON-file database (`db.json`). No npm
runtime deps, no build step.

**State at this point (approx):** core roles (student/org), opportunities, applications, hours,
awards, plans, leaderboard — the feature surface documented in `docs/FEATURES.md`.

## 2026-07-03 — Guardian consent flow + in-memory indexes  [reconstructed]

Committed as `8a1e4fe` ("Add guardian consent flow for minors and in-memory DB indexes", 12 files,
+1154/−80).

**WHAT:** (1) A guardian-consent gate for users under 18; (2) an in-memory index layer over the JSON
store.

**WHY (consent):** An `/llm-council` review (2026-07-02) concluded that verified guardian consent for
minors is a *precondition* for launch, not a feature — more urgent than any growth idea, because the
platform serves students and some are minors. Recorded as ADR-0010.

**HOW (consent):** Age is recomputed live from `dob` at every gate check (not cached at
registration), so 18+ users are never gated and a pending minor who turns 18 is unblocked
automatically with no background job. Single choke-point at `apply` (which transitively gates
messaging/check-in/endorsements, since those require an approved application), plus defense-in-depth
directly on those endpoints. Consent tokens are hashed (sha256) and never returned by any API — only
emailed. Public flow at `#consent/<token>` (approve/decline) and `#consent-manage/<token>` (revoke —
the kill switch). Correction mid-design: an earlier cut assumed *all* students were minors; that was
wrong (18+ students exist) and was fixed by making the gate age-dynamic and per-account.

**WHY (indexes):** Every lookup was an O(n) `Array.find`/`filter` over the JSON arrays; hot handlers
did several per request, and org dashboards/exports did N+1 scans. Pure waste since the keys are
known. Recorded as ADR-0011.

**HOW (indexes):** A lazily-built `IDX()` layer — `Map`s keyed by primary key and common foreign
keys — rebuilt on the *same* `_cacheVersion` invalidation the read cache already uses, so it can
never be staler than the last `saveDB()`. Invariant: never mutate the array `idxList()` returns
(shared bucket / frozen empty singleton) — spread first. `findIndex` write-path sites stay linear
scans (they need the array position). Covered by `test/index.test.js`.

## 2026-07-03 ~13:00 — Frontend editorial restyle ("less like AI")  [reconstructed]

**WHAT:** Reworked the SPA's visual language away from the generic-AI-landing-page look toward an
"editorial / restrained" aesthetic. CSS-only; no markup or logic changed.

**WHY:** Evan's read was that the UI, while polished, wore every AI-template signal at once (135°
gradient buttons, motion on everything, uppercase eyebrow tags, oversized radii). Those signals
undercut a portfolio piece meant to read as human-crafted. Direction chosen via a clarifying
question: **editorial / restrained** (over warm-handcrafted, Swiss-minimal, or a light touch-up).

**HOW:** Token-level changes so they propagate: flattened all `135deg` green/gold gradients to solid
fills; global radius `12px → 6px` and pulled oversized surfaces (18–24px) down to 8–12px; replaced
tinted glowy shadows with a neutral two-layer shadow; removed gratuitous motion (pulsing "live" dot,
floating hero blobs, support-banner shimmer, status-dot pulse, empty-state bounce, and the universal
hover-lift/scale/slide on cards/buttons/rows); de-shouted the `.sec-tag` eyebrows (dropped uppercase,
letter-spacing, and the tick line). Kept the genuinely bespoke touches: Fraunces + DM Sans, the
green/gold brand, the noise-texture overlay, numbered how-it-works steps.

**BUG / limitation hit:** `preview_screenshot` consistently timed out (renderer hang) even though the
page loaded `complete` with zero console errors. Root cause was environmental (renderer), not the
CSS. Worked around it by verifying every change through **computed styles** (`getComputedStyle` /
`preview_inspect`) instead of screenshots — which is the more reliable check for color/radius/shadow
anyway.
